# ユニットテスト実装

## 概要

プロジェクト全体にユニットテストが存在しないため、テストスイートを整備する。
Vitest + React Testing Library の環境はすでに構成済みだが、`@testing-library/jest-dom` のセットアップと実際のテストファイルが不足している。
あわせて、今後の実装でもテストを書くよう CLAUDE.md を更新する。

## 作業内容

### 新規作成ファイル

- `vitest.setup.ts` — jest-dom マッチャーのセットアップ
- `lib/utils.test.ts` — `cn()` ユーティリティ関数のテスト
- `app/(main)/team/actions.test.ts` — Server Actions（バリデーション・DB エラーハンドリング）のテスト
- `components/team/TeamCard.test.tsx` — 表示コンポーネントのレンダリングテスト
- `components/team/MemberList.test.tsx` — メンバーリストの表示・権限制御テスト
- `components/team/JoinTeamDialog.test.tsx` — チーム参加ダイアログのテスト
- `components/team/EditTeamNameDialog.test.tsx` — チーム名編集ダイアログのテスト

### 変更ファイル

- `vitest.config.mts` — `setupFiles` を追加して jest-dom を読み込む
- `package.json` — `@testing-library/jest-dom` を devDependencies に追加
- `CLAUDE.md` — 実装時のユニットテスト義務化ルールを追加

## 実装メモ

- `@testing-library/jest-dom` は vitest 用のセットアップが必要（`@testing-library/jest-dom/vitest` を import）
- `next/navigation`（useRouter）・`next/link` は vitest で `vi.mock()` が必須
- Server Actions のテストでは `@/lib/supabase/server` の `createClient` と `next/cache` の `revalidatePath` をモック
- コンポーネントが Server Actions を呼ぶ場合は `@/app/(main)/team/actions` をモック
- コンポーネントが直接 Supabase クライアントを呼ぶ場合（MemberList・PlayerForm）は `@/lib/supabase/client` をモック
- テストファイルはテスト対象ファイルと同じディレクトリに配置する
