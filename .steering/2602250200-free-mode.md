# フリーモード機能の実装

## 概要

チーム未所属でも試合を記録できる「フリーモード」機能を追加する。任意の2チーム名を入力して試合を記録でき、結果は通算成績に含めない。

## 技術的な注意点

### DB
- `created_by` のバックフィルは `games.team_id → teams.owner_id` を使用
- 子テーブルの RLS は `game_id IN (SELECT id FROM games WHERE ...)` パターンなので、games のポリシー変更が自動的に伝播
- 通算成績ビューは `JOIN players p ON p.id = l.player_id`（INNER JOIN）で未登録選手が自動排除されるが、安全策として `WHERE g.is_free_mode = false` も追加

### フロントエンド
- トップページ (`page.tsx`) は **今日の試合** のみ表示、`TodayGameCard` を使用
- 試合一覧ページ (`games/page.tsx`) は **全試合履歴** を表示、`GameCard` を使用
- 試合詳細ページは Server Component (`page.tsx`) → Client Component (`GameDetailClient.tsx`) の構成
- `spectate/page.tsx` は `games/[id]` へのリダイレクトのみ（変更不要）
- フリーモード時は `is_home` の概念がない → home/visitor は入力順で決定
- ラインナップは両チームとも手動入力（`players` prop なし）
- 選手交代ダイアログはベンチ選手リスト不要、テキスト入力のみ
