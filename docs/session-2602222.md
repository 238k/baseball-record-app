# 作業レポート 2026-02-22

## 1. DH制の試合詳細画面 — 投手表示の修正

**問題**: DH制の試合で、投手とDH打者が同じ `batting_order` を持つため React の key 重複エラーが発生していた。

**修正内容** (`app/(main)/games/[id]/page.tsx`):

- `use_dh` フラグをクエリに追加し、DH制かどうかを判定
- DH投手（`position=投` かつ同じ `batting_order` にDHエントリがある行）をラインナップテーブルから分離
- `LineupTable` コンポーネントに先発投手の別枠表示を追加
- 9番打者と投手の間に `border-t-2` で罫線を表示
- 投手行は「先発 / 名前 / 投」のカラムレイアウトで表示

**コミット**: `6d6ce76`

## 2. カーソル表示の修正

**問題**: Tailwind CSS v4 の Preflight が `button` や `a` に `cursor: pointer` を設定しなくなり、ホバー時にカーソルが矢印のままだった。

**修正内容** (`app/globals.css`):

- `@layer base` に `button, a, select, summary, [role="button"]` への `cursor: pointer` を追加

**コミット**: `be5dc03`

### 実装した機能一覧

- **打席結果入力**: 全17種の結果ボタン（1B, 2B, 3B, HR, BB, IBB, HBP, K, KK, GO, FO, LO, DP, SF, SH, FC, E）
- **確認ダイアログ**: 「田中 → 単打 でよいですか？」形式
- **走者・得点入力ダイアログ**: 各走者の進塁先をデフォルト値付きで表示し、手動修正可能。打点も自動計算＋手動修正
- **次打者自動移動**: 結果確定後に打順を自動で進める（9番の次は1番に戻る）
- **攻守交代**: 3アウト到達時にダイアログ表示、イニング進行
- **投手交代**: `pitching_records` の `inning_to` 更新 + 新規レコード作成
- **試合終了**: `games.status` を `finished` に変更
- **状態復元**: ページリロード後も `useGameState` が at_bats から試合状態を再構築

### DB書き込みフロー（1打席あたり）

1. `at_bats` に INSERT（inning, inning_half, batting_order, lineup_id, result, rbi）
2. `base_runners` に INSERT（打席開始時の走者スナップショット）
3. `runner_events` に INSERT（scored / out イベント。本塁打の打者自身も scored に含む）
