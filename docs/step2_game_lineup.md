## 実装仕様

### トップ画面 `/`（更新）

所属チームの試合一覧を表示：
- 試合日・相手チーム名・ホーム/ビジター・スコア（終了試合のみ、`v_scoreboard` から集計）・ステータス
- ステータスバッジ：`scheduled`=「試合前」、`in_progress`=「試合中」、`finished`=「終了」
- 「新規試合登録」ボタン
- 試合カードをタップすると試合詳細へ

### 試合登録 `/games/new`

入力項目：
- 相手チーム名（必須）
- 試合日（必須・デフォルト今日）
- 場所
- ホーム/ビジター（トグル）
- イニング数（デフォルト9・草野球で7回制の場合は手動変更）
- DH制（なし/あり トグル、デフォルト: なし）

登録後はオーダー登録画面へ遷移。

### オーダー登録 `/games/[id]/lineup`

**DH制の切り替え：**

オーダー登録画面のヘッダーに DH制トグル（なし/あり）を表示。切り替えると `games.use_dh` を更新し、ポジションが自動変換される（投 ↔ DH）。

**自チームのオーダー：**

打順1〜9にデフォルトポジション（通常: 投・捕・一・二・三・遊・左・中・右、DH制: DH・捕・一・二・三・遊・左・中・右）がプリセットされた状態で表示。
- 選手：登録選手からセレクト（背番号・名前表示）または「未登録選手」を選んで名前を直接入力
- 守備位置：固定ラベル表示（ドラッグ&ドロップで打順を変更）
- ドラッグハンドル（GripVertical）で行を掴んで打順を入れ替える（@dnd-kit 使用）

**DH制の場合の投手指定：**

DH制の場合、先発投手選択UIを別途表示。自チームは登録選手セレクト + 未登録選手対応、相手チームは名前入力。

**相手チームのオーダー：**

デフォルトでは折りたたまれて非表示。展開ボタンで入力欄を表示。
打順1〜9まで名前のみ入力（全員未登録扱い）。
入力省略可能（省略した場合は「相手選手1」などのプレースホルダーで記録）。

**投手の初期設定：**

通常モード: ポジション「投」の打者を自動的に先発投手として判定。
DH制: 別途選択した投手を先発として登録。
`pitching_records` にレコードを作成（inning_from=1）。

**保存後の動作：**

「記録を開始する」ボタンを表示。タップすると AlertDialog で確認後、試合ステータスを `in_progress` に変更し、試合詳細画面へ遷移。

**ページ離脱確認：**

未保存の変更がある場合、ブラウザ離脱時（タブ閉じ・リロード）にネイティブ確認ダイアログ、「試合詳細に戻る」クリック時に AlertDialog を表示。

### 試合詳細 `/games/[id]`（このステップでは簡易版）

- 試合情報（日付・相手・場所・イニング数）
- オーダー表（自チーム・相手）※相手チームが全員プレースホルダーの場合は非表示
- 「記録を入力する」ボタン（`in_progress`のみ表示）
- 「オーダーを編集する」ボタン（`scheduled`のみ表示）

## UI・UX要件

- タブレット（iPad）での操作を主に想定
- shadcn/uiのコンポーネントを積極的に使用すること
- 選手セレクトはshadcn/uiの `Select` を使用、背番号順でソート
- 守備位置は日本語略称で表示（投・捕・一・二・三・遊・左・中・右・DH）
- オーダーの打順変更はドラッグ&ドロップ（@dnd-kit/core, @dnd-kit/sortable）
- 入力中のデータはページ離脱時に確認ダイアログ（`beforeunload` + shadcn/uiの `AlertDialog`）を表示
- ボタンは `size="lg"` を基本とすること
