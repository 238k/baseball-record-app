## 実装仕様

### 記録入力画面 `/games/[id]/input`

#### 画面レイアウト（タブレット縦向き想定）

shadcn/uiの `Card`・`Button`・`Dialog`・`AlertDialog` を使用してください。
打席結果ボタンは `size="lg"` で `min-h-16 w-full text-lg` を基本とし、タップしやすいサイズにすること。
確認ダイアログ・走者入力ダイアログはshadcn/uiの `AlertDialog` または `Dialog` を使用。

```
┌─────────────────────────────────────┐
│  自チーム 3 - 2 相手   3回表        │  ← ヘッダー
├─────────────────────────────────────┤
│  ●●○  アウト                        │  ← アウトカウント（●=アウト済）
│  [1塁] [2塁] [3塁]                  │  ← 走者（走者ありは強調表示）
├─────────────────────────────────────┤
│  打者: #3 田中  （3番・左翼）        │
├────────────────┬────────────────────┤
│   安打系        │   アウト系          │
│  [単打]         │  [ゴロ]            │
│  [二塁打]       │  [フライ]          │
│  [三塁打]       │  [ライナー]        │
│  [本塁打]       │  [三振(空)]        │
│                │  [三振(見)]        │
├────────────────┼────────────────────┤
│   出塁系        │   その他            │
│  [四球]         │  [犠打]            │
│  [死球]         │  [犠飛]            │
│  [エラー]       │  [併殺打]          │
│  [FC]          │                    │
└────────────────┴────────────────────┘
```

#### 打席結果ボタンを押した後のフロー

1. **確認ダイアログ**を表示（「田中 → 単打 でよいですか？」）
2. 確認後、**走者・得点入力ダイアログ**を表示
3. ダイアログで以下を設定：
   - 各走者がどこに進んだか（進塁先 or 得点 or アウト）
   - 打者自身の進塁先（単打なら1塁固定だが、エラーなら変わることも）
   - 打点の数（自動計算を表示しつつ手動修正可能）
4. 確定ボタンで `at_bats`・`base_runners`・`runner_events` を保存
5. アウト数が3になったら**攻守交代ダイアログ**を表示してイニングを進める

#### 走者・得点入力ダイアログの仕様

打席結果ごとのデフォルト動作（変更可能）：

| 結果 | 打者 | 1塁走者 | 2塁走者 | 3塁走者 |
|------|------|---------|---------|---------|
| 単打 | →1塁 | →2塁 | →3塁 | 得点 |
| 二塁打 | →2塁 | →3塁 | 得点 | 得点 |
| 三塁打 | →3塁 | 得点 | 得点 | 得点 |
| 本塁打 | 得点 | 得点 | 得点 | 得点 |
| 四球/死球 | →1塁 | →2塁(1塁いた時) | そのまま | そのまま |
| ゴロ/フライ等 | アウト | そのまま | そのまま | そのまま |
| 犠打 | アウト | →2塁 | →3塁 | 得点 |
| 犠飛 | アウト | そのまま | そのまま | 得点 |
| 併殺打 | アウト | アウト | そのまま | そのまま |

得点が発生した場合は自動的に `runner_events`（event_type='scored'）を作成。

#### 投手交代

入力画面のメニューから「投手交代」を選択：
- 新しい投手をオーダーから選択
- 現在の `pitching_records` の `inning_to` を現在のイニングに更新
- 新しい `pitching_records` を作成

#### 試合終了

最終イニング終了後、「試合を終了する」ボタンを表示。
タップすると `games.status` を `finished` に変更。

## useGameState フックの仕様

```typescript
// このフックで管理する状態
type GameState = {
  game: Game
  currentInning: number
  currentHalf: 'top' | 'bottom'
  currentOuts: number
  currentBatterOrder: number
  baseRunners: {
    first: Lineup | null
    second: Lineup | null
    third: Lineup | null
  }
  score: {
    home: number
    visitor: number
  }
}
```

Supabaseから最新の `at_bats` と `base_runners` を取得して状態を構築する。
